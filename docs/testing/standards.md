# テスト品質基準

## 必須要件

- **ユニットテスト**: カバレッジ80%以上
- **統合テスト**: 主要フロー実装
- **テスト実行**: CI/CDで自動化

## 推奨要件

- **ミューテーションテスト**: 品質スコア70%以上
- **パフォーマンステスト**: 重要機能で実装
- **アクセシビリティテスト**: WCAG 2.1 AA準拠

## テスト実装要件（コミット前に必ず実施）

- **テストツール**: テスト・動作確認の際はplaywrightのMCPツールを使用して動作確認を行うこと
- **テスト実施タイミング**: テスト・動作確認は修正を行った際は必ず行うこと
- **E2Eテスト**: E2Eテストとしてユーザ目線での動作が問題ないかしっかりと確認すること

## コード品質基準

### 必須要件

- TypeScript型チェック: エラー0件
- ESLint: 警告・エラー0件
- Prettier: フォーマット統一
- 関数・クラス: 適切な型定義
- コメント: 複雑なロジックに説明

### 推奨要件

- 関数サイズ: 50行以下
- クラスサイズ: 300行以下
- 循環的複雑度: 10以下
- 型カバレッジ: 95%以上
