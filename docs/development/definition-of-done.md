# Definition of Done チェックリスト

## 開発段階

- [ ] 要件定義に沿った機能実装完了
- [ ] TypeScript型チェック通過
- [ ] ESLint・Prettier適用済み
- [ ] 単体テスト実装・通過（カバレッジ80%以上）
- [ ] 統合テスト実装・通過（必要に応じて）

## コードレビュー段階

- [ ] プルリクエスト作成・テンプレート記入
- [ ] 自動テスト（CI）通過
- [ ] コードレビュー実施・承認取得
- [ ] セキュリティチェック完了
- [ ] パフォーマンステスト通過（必要に応じて）

## 🚀 デプロイ段階（Production一本運用）

- [ ] ローカルでの十分な動作確認・テスト
- [ ] 静的解析通過（TypeScript + ESLint）
- [ ] 本番環境への直接デプロイ
- [ ] **デプロイ後の即座確認・監視**
- [ ] 問題発見時の迅速な修正対応

## 修正時の注意点（必須遵守）

1. **影響範囲の確認**: 該当修正によって他の処理に問題がないか慎重に確認を行って作業すること
2. **既存機能の保護**: 他の動作に関しても修正が必要な場合は既存の期待値の動作が正常に起動するように修正すること
