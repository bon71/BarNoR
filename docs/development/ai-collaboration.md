# 他AIツールとの連携

## 🤝 AI実装ワークフローにおけるClaudeCodeの役割

- **主責務**: 複雑なロジック実装、アーキテクチャ設計、コードレビュー、実装プロンプト作成
- **対象タスク**: バックエンドAPI実装、複雑なフロントエンド機能、データベース設計・実装、セキュリティ機能実装、エラーハンドリング・例外処理
- **連携**: Cursor（コーディング実装）、GitHub Copilot（レビュー・デバッグ）、v0.dev（UI実装）との協調

## Cursor（コーディング実装担当）との連携

### Cursorの責務
ClaudeCodeが作成した実装プロンプトに基づくコーディング実装

### 対象タスク
- ClaudeCodeが作成したPRPファイル（Phase 1, Phase 2, Phase 3など）に基づくコード実装
- テストコード生成・実装
- 定型的なCRUD操作実装
- データマイグレーション
- 設定ファイル生成

### 連携方法
1. ClaudeCodeがPRPファイル（実装プロンプト）を作成
2. PRPファイルをCursorで開き、プロンプト内容をCursorに渡して実装
3. 実装完了後、ClaudeCodeがコードレビュー・品質チェック

### Cursorファイル確認
コードに変更を加えたり新規で作成した際はCursorで変更後の該当ファイルを確認するため、`cursor`コマンドで該当ファイルを開く：

```bash
cursor sample.ts
```

## GitHub Copilot（レビュー・デバッグ担当）との連携

### 責務
コード品質向上、バグ発見・修正支援

### 対象タスク
- コードレビュー・改善提案
- デバッグ支援・エラー解析
- リファクタリング提案
- パフォーマンス最適化

### 連携方法
- 実装後のコードレビュー
- デバッグ時の支援要請

## v0.dev（UI実装担当）との連携

### 責務
デザインシステムに基づく初期のUI実装

### 連携方法
- Markdownドキュメント提供
- 実装コードの統合
